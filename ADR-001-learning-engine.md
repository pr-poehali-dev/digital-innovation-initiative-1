# ADR-001: Learning Engine Architecture

**Статус:** accepted  
**Дата:** 2026-06-14

## Контекст

Goal Builder (1.2) строит план развития, но не даёт инструмента для реального обучения.  
Пользователь видит шаги «что изучать», но не знает «на чём» и не может начать прямо сейчас.

## Решение

Строим Learning Pack как **retrieval-first AI learning engine**:

> Мы строим не генератор советов, а слой персонализированной навигации поверх доверенного корпуса материалов.

### 4 зафиксированных принципа

| # | Принцип | Что это означает |
|---|---------|-----------------|
| 1 | **Retrieval-first hybrid** | Рекомендации = реально найденные материалы, а не LLM-генерация «из головы» |
| 2 | **Material = first-class entity** | Материал — отдельная сущность в БД с URL, доменом, trust_level, provenance |
| 3 | **AI = post-processing layer** | AI только ранжирует, объясняет и персонализирует найденное, не является источником |
| 4 | **Learning pack = async job** | Подбор материалов — фоновая операция; результат кешируется |

### Архитектурные слои

```
[Goal → Milestone]
      ↓
[Retrieval Engine]        ← DuckDuckGo + whitelist-домены
      ↓
[Material Catalog]        ← shared table, переиспользуется между пользователями
      ↓
[AI Enrichment]           ← relevance score, reason, summary (кешируется)
      ↓
[User Learning Pack]      ← milestone_materials + user_material_progress
```

### Trust levels (content_sources)

| Уровень | Тип | Примеры |
|---------|-----|---------|
| **A — Официальный** | Госорганы, стандарты, регуляторы | cbr.ru, consultant.ru, digital.gov.ru |
| **B — Проверенный** | Университеты, крупные платформы, медиа | hse.ru, stepik.org, habr.com |
| **C — Открытый** | Статьи, блоги, видео | wikipedia.org, youtube.com |

### Что AI делает

- Строит поисковые запросы по milestone + goal
- Из найденных результатов отбирает лучшие (relevance score)
- Объясняет почему материал выбран (selection_reason)
- По запросу пользователя генерирует краткую выжимку (кешируется в material_summaries)

### Что AI НЕ делает

- Не придумывает URL
- Не является источником знаний
- Не заменяет первоисточники

## Последствия

**Новые таблицы:** content_sources, materials, material_summaries, milestone_materials, user_material_progress, learning_jobs

**Пользовательский поток:**
```
Паспорт → Цель → Gap-анализ → План → Материалы → Выжимка → Изучение → Прогресс
```

**Что будет строиться следующим:**  
AI-Coach (чат по теме шага) → Gap re-evaluation после прогресса → Competency confirmation

## Варианты которые не выбрали

| Вариант | Причина отказа |
|---------|---------------|
| Pure LLM («придумай 5 материалов») | Галлюцинации ссылок, нет повторяемости, нет доверия |
| Только ручной каталог | Медленно масштабируется, дорого наполнять |
| Только live web-search | Нестабильные результаты, трудно контролировать качество |
