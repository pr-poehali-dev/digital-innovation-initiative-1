## 0. Главная идея

DocMind AI — не «кабинет с задачами», а **персональная операционная система развития человека**.

Платформа отвечает пользователю на 5 главных вопросов:

1. **Что я уже знаю** и чем это подтверждено?
2. **Чего мне не хватает** для моей цели?
3. **Что мне учить дальше** и в каком порядке?
4. **Какие курсы, книги, программы** мне реально подходят?
5. **Как я расту по факту**, а не по ощущению?

---

## 1. Что должна делать платформа

### 1.1 Фиксировать образовательный бэкграунд
- Дипломы, сертификаты, удостоверения
- Пройденные курсы и программы
- Изученные учебные материалы (книги, лекции, конспекты, презентации)
- Учебные работы пользователя

### 1.2 Строить образовательные и карьерные цели
- «Хочу стать юристом»
- «Хочу подтянуть английский до B2»
- «Хочу освоить гражданское право»
- «Готовлюсь к магистратуре»

### 1.3 Давать персональный план развития
- Карта компетенций (что есть / чего нет)
- Траектория обучения (этапы, навыки, материалы)
- Рекомендации курсов / книг / программ

### 1.4 Показывать реальный прогресс
- По знаниям
- По навыкам
- По дисциплине
- По зонам роста

### 1.5 Помогать AI-наставником
- Разбор материалов
- Поиск курсов и ресурсов
- Перестроение плана при срывах
- Объяснение «почему именно этот шаг»

## 2. Трёхуровневая модель доказательств знаний

### Уровень 1 — Formal Evidence (документ)
- Диплом / сертификат / курс / удостоверение
- **Высокий уровень доверия**

### Уровень 2 — Learning Exposure (материал)
- Загруженная книга / лекция / методичка / презентация
- Тема **знакома**, но mastery под вопросом

### Уровень 3 — Proven Competence (доказано действием)
- Тест / проект / выполненная работа / артефакт
- **Самый сильный** тип подтверждения

**Принцип:** Система НИКОГДА не путает «изучал» и «владею». Каждое знание имеет источник и confidence score.

## 3. Продуктовые модули (8 штук)

### 📜 Модуль 1. Educational Passport — Паспорт образования
- Дипломы, сертификаты, курсы, удостоверения
- Загрузка → OCR / AI-анализ → извлечение названия, учреждения, направления, дисциплин, компетенций
- Ручное подтверждение пользователем
- Хранение: что подтверждено, что предположено AI, что подтвердил пользователь

### 🧠 Модуль 2. Personal Knowledge Graph — Карта знаний
- Что уже знает (источники: дипломы, материалы, проекты)
- В каких темах база сильная
- Где знания поверхностные
- Где пробелы

### 🎯 Модуль 3. Goal Builder — Конструктор целей
- Профессия / направление / навык / самостоятельное изучение
- Система: определяет целевые компетенции → сравнивает с картой → gap analysis → персональная траектория

### 📚 Модуль 4. Recommendation Engine — Подбор обучения
- Курсы, образовательные программы
- Книги, учебники, статьи
- Видео, лекции, практические задания
- Фильтры: цель / уровень / бюджет / язык / формат / срок / страна / бесплатное-платное

### 📊 Модуль 5. Progress Dashboard — Дашборд развития
**Главный экран кабинета:**
- Главная цель + следующий шаг
- Что делать сегодня (3 действия + 1 учебный блок + 1 повторение)
- Образовательный профиль (счётчик документов, ключевые области)
- Активные траектории (% прогресса, риски, дедлайны)
- Зоны роста
- AI-совет «куда направить усилия»
- Динамика 7/30/90 дней

### 🤖 Модуль 6. AI-наставник
Не чат, а помощник по развитию:
- Объясняет текущий уровень
- Находит пробелы
- Составляет маршрут
- Подбирает ресурсы
- Делает weekly review
- Аргументирует «почему именно этот шаг»

### ⚠️ Модуль 7. Growth Zones — Зоны роста
- Где отстаёшь
- Какие навыки недокачаны
- Какие темы отсутствуют
- Где нужна практика для подтверждения

### 🧬 Модуль 8. Brain & Learning Performance (продвинутый)
- Внимание, память, концентрация
- Глубина понимания, устойчивость
- Регулярность, когнитивная нагрузка

## 5. Backend / Data model — новые сущности

```
users
education_records        # диплом, сертификат, курс
education_record_files   # файлы документов
institutions             # вузы, школы, платформы
learning_materials       # книги, лекции, методички
learning_material_analysis  # AI-анализ материала
competencies             # справочник компетенций
user_competencies        # компетенции пользователя
competency_evidence      # доказательства (документ/материал/проект/тест)
goals                    # цели пользователя
learning_paths           # траектории обучения
milestones               # этапы траекторий
recommendations          # подобранные курсы/книги/материалы
progress_snapshots       # снимки прогресса по датам
assessments              # тесты / экзамены
weekly_reviews           # еженедельные ревью AI
```

Для каждого знания/навыка хранить:
- название, область, уровень
- источник, тип доказательства
- confidence score
- кем подтверждено (AI / пользователь / документ / тест / проект)

## 6. AI — 5 ролей

1. **Анализ документов** — извлечение из дипломов, сертификатов, материалов
2. **Извлечение навыков и тем** — что покрывает материал, что заявлено vs подтверждено
3. **Gap Analysis** — сравнение профиль ↔ цель
4. **Персональные рекомендации** — курсы / книги / упражнения / маршруты
5. **AI Review** — еженедельно: рост / застой / куда фокус

---

## 7. Принципы реализации (важно!)

### 7.1 «Изучал» ≠ «владею»
Если пользователь загрузил книгу — это **не** mastery. Хранить confidence.

### 7.2 Все AI-выводы explainable
Если AI говорит «у тебя есть база» — должен показать на основании каких документов/проектов это выведено.

### 7.3 Право пользователя на корректировку
AI ошибается. Юзер всегда может подтвердить / отклонить / уточнить.

### 7.4 Прогресс доказательный
Не рисовать красивые цифры без основания.

## 8. Сценарий: «Хочу стать юристом»

1. **Создаётся цель** в Goal Builder
2. **Целевые компетенции:** правовая логика, отрасли права, работа с правовыми текстами, письменная аргументация, анализ кейсов и норм
3. **Сравнение с профилем:**
   - Что уже есть (из дипломов/материалов)
   - Чего не хватает
   - Что можно засчитать
4. **Предложение:**
   - Программы обучения
   - Курсы, книги, учебники
   - Практические задания
   - План на 3 / 6 / 12 месяцев
5. **Отслеживание прогресса** + обновление карты развития

## 9. Что НЕ должна делать платформа

- Не считать загруженный файл = освоенной темой
- Не рисовать прогресс из воздуха
- Не выдавать рекомендации без объяснения
- Не игнорировать ручные правки пользователя
- Не нарушать изоляцию данных (Voronov ≠ другой юзер)
