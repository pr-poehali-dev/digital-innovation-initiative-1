## 🔥 ЭТАП 0.5 — P0: Управляемое использование документов (СЕЙЧАС)

**Проблема:** AI берёт основные вехи из образца презентации и игнорирует стандарт IPMO, хотя в промпте это указано. Документы воспринимаются «плоско», без приоритетов и режимов использования.

**Решение:** structured orchestration — система понимает назначение каждого документа.

### 0.5.1 БД: расширить task_documents ✅
- [x] Добавить `usage_mode` (как использовать)
- [x] Добавить `priority` (high/medium/low)
- [x] Добавить `must_use` (обязательный документ)
- [x] Добавить `instruction` (текстовая инструкция «что брать / что не брать»)
- [x] Расширить `role` enum: standard / methodology / template / content / background

### 0.5.2 Справочник ролей ✅
- [x] `standard` — нормативный стандарт (IPMO, ГОСТ, регламент) — высший приоритет
- [x] `methodology` — методичка / guideline (логика, подход)
- [x] `template` — шаблон формата (брать ТОЛЬКО формат, НЕ содержание)
- [x] `content` — содержательный источник (факты, тезисы)
- [x] `background` — фоновый материал (только контекст)

### 0.5.3 UI: NewTaskPage с детальной настройкой ✅
- [x] Для каждого документа выбор роли
- [x] Для каждого документа `usage_mode` (зависит от роли)
- [x] Чекбокс «обязательный»
- [x] Поле «инструкция» (что брать / не брать)
- [x] Селектор приоритета (высокий / средний / низкий)

### 0.5.4 Backend generate: structured orchestration ✅
- [x] Шаг 1: классификация документов по ролям
- [x] Шаг 2: явная иерархия в системном промпте: **standard > content > methodology > template > background**
- [x] Шаг 3: для TEMPLATE — текст ограничен 2000 символов (только формат, без вех)
- [x] Шаг 4: явные правила конфликта
- [x] Шаг 5: explainability — карта влияния документов в конце ответа

### 0.5.5 Критерии приёмки — требуется e2e-проверка пользователем
- [ ] Взять структуру презентации из стандартов IPMO
- [ ] Взять содержимое из материалов пользователя
- [ ] Взять только формат из образца презентации (не содержание!)
- [ ] НЕ копировать содержательные вехи из чужого образца
- [ ] После генерации показать карту: «структура → IPMO, контент → диплом, формат → образец»
