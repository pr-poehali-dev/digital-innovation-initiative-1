# Stage 14 — Problems without solution queue

## Что это
Stage 14 закрывает последний разрыв в воронке трансформации: проблема, у которой уже есть гипотеза, но она ещё не привязана ни к одному решению или системе.

## Критерий preset
`without_solution` — проблема считается «без решения», если:
- у неё **нет** привязанного решения/системы, **и**
- у неё **есть** хотя бы одна гипотеза.

Это сознательно уже фильтра «нет решения вообще» — чтобы не пересекаться со Stage 13 (`without_hypothesis`). У каждой очереди — одна причина и одно следующее действие:
- `without_hypothesis`: проблема ещё не осмыслена → создать гипотезу
- `without_solution`: проблема уже осмыслена → довести до решения

Виджет **«Проблемы без решения»** на overview считает по этой же формуле, поэтому число на виджете и счётчик в очереди всегда совпадают.

## Что изменилось
- Виджет **«Проблемы без решения»** ведёт на `tab=pains&preset=without_solution`
- На вкладке **«Проблемы»** preset-режим показывает бейдж **«Нет решения»**
- Контекстный CTA на карточке:
  - если в проекте есть хотя бы одно решение — **«Привязать к решению»** (открывает уже существующий инлайн-редактор привязки)
  - если решений в проекте нет вообще — **«Создать решение»** (fallback, ведёт на вкладку «Решения и системы»)
- Fallback сохраняет контекст возврата: на вкладке «Решения и системы» показывается баннer-подсказка с кнопкой «Вернуться к очереди»
- После успешной привязки решения из очереди: success-feedback, подсветка следующей карточки, либо completion-state **«Все проблемы привязаны к решению»**

## Out of scope
- Автоматическая привязка только что созданного решения к проблеме
- Мультишаговый guided wizard «создал → сразу выбрал → связал»

Пользователь сам возвращается к проблеме и сам выбирает нужное решение — это осознанный компромисс, чтобы не раздувать scope этапа.

## Demo path
1. На overview кликнуть **«Проблемы без решения»**
2. Открывается вкладка «Проблемы» с preset `without_solution`, виден бейдж «Нет решения»
3. Если решения есть — нажать «Привязать к решению», выбрать из списка, сохранить
4. Если решений нет — нажать «Создать решение», создать его на вкладке «Решения и системы», вернуться по кнопке «Вернуться к очереди»
5. После сохранения — success-feedback и переход к следующей карточке, либо completion-state
